Video_XH - ein CMSimple_XH-Plugin

Video_XH ermöglicht die Präsentation von Videos auf einer CMSimple_XH-Website. Es verwendet wo immer möglich das HTML5 Video-Element, und nutzt andernfalls den Flash-Player. Die unterstützten Video-Formate hängen vom Browser bzw. dem Flash-Player ab. Für das beste Nutzungserlebnis sollte JavaScript im Browser des Besuchers aktiviert sein.

Voraussetzungen

Video_XH ist ein Plugin für CMSimple_XH. Es benötigt eine UTF-8 kodierte Version, und PHP ≥ 5.4.0 mit der JSON Extension.

Installation

The Installation erfolgt wie bei vielen anderen CMSimple_XH-Plugins auch. Im CMSimple_XH-Wiki finden Sie weitere Informationen.

  1. Sichern Sie die Daten auf Ihrem Server.
  2. Entpacken Sie das herunter geladene Archiv auf Ihrem Computer.
  3. Laden Sie das komplette Verzeichnis video/ auf Ihren Server in das CMSimple_XH-Pluginverzeichnis hoch.
  4. Machen Sie die Unterverzeichnisse config/, css/ und languages/ beschreibbar.
  5. Gehen Sie zu "Video" im Administrationsbereich, um zu prüfen, ob alle Voraussetzungen erfüllt sind.

Einstellungen

Die Plugin-Konfiguration wird wie bei vielen anderen CMSimple_XH-Plugins auch im Administrationsbereich durchgeführt. Wählen Sie "Plugins" → "Video".

Sie können die Voreinstellungen von Video_XH unter "Konfiguration" ändern. Hinweise zu den Optionen werden angezeigt, wenn Sie die Hilfe-Icons mit der Maus überfahren.

Die Lokalisierung wird unter "Sprache" vorgenommen. Sie können dort die Sprachtexte in Ihre eigene Sprache übersetzen (falls keine entsprechende Sprachdatei zur Verfügung steht), oder diese gemäß Ihren Wünschen anpassen.

Das Aussehen von Video_XH kann unter "Stylesheet" angepasst werden. Um das Aussehen des Players zu ändern, sollten Sie einen anderen Skin verwenden.

Verwendung

Da Video_XH keinen Upload anbietet, müssen Sie Ihre Video-Dateien in den eingestellten Order per FTP, Filebrowser oder Uploader_XH hoch laden. Es werden die folgenden Video-Formate unterstützt: MP4 (AVC Baseline@L3.0/AAC), WebM (VP8/Vorbis) und OGG (Theora/Vorbis). Sie können Ihre Videos mit XMedia Recode (verwenden Sie das "HTML 5" Profil) leicht in diese Formate konvertieren. Um alle bedeutenden Browser zu unterstützen sollten Sie eine *.mp4 und eine *.webm/*.ogv Version hoch laden. Zusätzlich können Sie ein so genanntes Poster in den gleichen Ordner hoch laden, also ein Bild im JPEG-Format (*.jpg), das angezeigt wird bevor das Video abspielt.

Um ein Video auf einer CMSimple_XH-Seite anzuzeigen, fügen Sie dort den folgenden Plugin-Aufruf ein:

{{{PLUGIN:video('%NAME%', '%OPTIONEN%');}}}

Die Bedeutung der Parameter:

%NAME%
Der Name der Video-Datei ohne Datei-Erweiterung. Um genau zu sein, ist dies ein Dateipfad relativ zum eingestellten Video-Ordner.
%OPTIONEN%
Optionen für dieses Video als Zeichenkette im selben Format wie ein Query-String (alles hinter dem Fragezeichen in einem HTTP-GET-Request). Optionen, die nicht angegeben werden, verwenden die entsprechenden Werte aus der Plugin-Konfiguration. Sie können den "Aufruf-Assitenten", der in der Plugin-Administration verfügbar ist, verwenden, um den Aufruf zusammen zu stellen. Nachdem Sie mit der Eingabe der gewünschten Werte fertig sind, kopieren Sie einfach den Inhalt der Textarea ganz unten und fügen ihn auf einer Seite ein.

Um ein Video auf allen Seiten anzuzeigen, fügen Sie das folgende ins Template ein:

<?php echo video('%NAME%', '%OPTIONS%');?>

In diesem Fall, oder wenn Sie Videos in einer Newsbox oder in einem anderen Plugin, das im Template aufgerufen wird, anzeigen wollen, müssen Sie die Konfigurationsoption "auto hjs" aktivieren.

Beispiele

{{{PLUGIN:video('LotR');}}}
Dies wird das LotR.webm, LotR.mp4 und/oder LotR.ogv Video im konfigurierten Video-Ordner mit der konfigurieren Breite und Höhe anzeigen. Wenn LotR.jpg existiert, wird es als Poster verwendet.
{{{PLUGIN:video('music/thriller', 'width=320&height=240');}}}
Wird das "thriller" Video aus dem Unterordner music/ mit einer Größe von 320px × 240px anzeigen. Beachten Sie, dass das Skalieren beim Playback nicht unbedingt die beste Option ist.
{{{PLUGIN:video('banner', 'autoplay&loop&controls=0');}}}
Wird automatisch das "banner" Video in einer Schleife abspielen, ohne Steuerelemente anzuzeigen. Das ist besonders nützlich für Banner- oder Hintergrundvideos.

Untertitel

In unterstützenden Browsern können die Video-Player optional Untertitel anzeigen. Die Untertitel müssen sich in separaten Dateien befinden (nicht in den Container gemuxt), wobei WebVTT das empfohlene Datei-Format ist, obgleich SRT ebenfalls funktionieren kann. Legen Sie einfach eine entsprechende Datei *.vtt bzw. *.srt neben den Video-Dateien ab, und der Besucher ist in der Lage die Untertitel zu aktivieren, wenn er das möchte.

Für mehrsprachige Websites können Sie Untertitel in den entsprechenden Sprachen nutzen. Benennen Sie sie einfach *_LANG.vtt bzw. *_LANG.srt, z.B. also *_en.vtt und *_de.vtt. Der Player wird nur diejenigen Untertitel der aktuell gewählten Sprache anbieten.

Skins

Der Video-Player von Video_XH kann mit Skins gestylt werden. Um einen anderen Skin als den Standard zu verwenden, müssen Sie eine Datei mein_skin.css im lib/ Ordner erzeugen, die CSS Regeln für .vjs-mein_skin-skin enthält, und Sie müssen die entsprechende Konfigurationsoption ändern. Die einfachste Art Ihren eigenen Skin zu erzeugen ist vermultich der Video.js Player Designer.

Fehlerbehebung

Falls das Video gar nicht abgespielt werden kann, prüfen Sie, ob dieses Video im Browser abgespielt werden kann, wenn Sie es direkt über die URL aufrufen. Das funktioniert allerdings nur, wenn der Browser das Video-Format nativ unterstützt und das Video mit dem richtigen MIME-Typ ausgeliefert wird. Sollte letzteres nicht der Fall sein, dann können Sie einen Apache-Server vermutlich mit den folgenden Zeilen in .htaccess entsprechend konfigurieren:

AddType video/webm .webm
AddType video/mp4 .mp4
AddType video/ogg .ogv

Wenn dies nicht möglich ist, bitten Sie Ihren Provider oder Server-Administrator die nötige Konfiguration vorzunehmen.

Wenn das Video nicht sofort abgespielt werden kann, liegt das vermutlich an einem Kodierungs- oder Multiplex-Fehler. Es scheint, dass viele MP4-Muxer das "moov" Atom ans Ende des Videos hängen, was den Download der gesamten Datei erforderlich macht, bevor die Wiedergabe starten kann. Das Beste ist zu versuchen, das Video mit Yamb (MP4) oder MKVToolNix (WebM) erneut zu multiplexen.

Einschränkungen

  • Das HTML5 Video Element ist in HTML 4.01 und XHTML 1.0/1.1 ungültig. Falls Sie es für nötig halten, sollten Sie also die Doctype-Deklaration in Ihrem Template ändern:

    <!DOCTYPE html>

    VideoJS' CSS validiert nicht. Grundsätzlich ist es CSS 3, aber für best mögliche Cross-Browser Interoperabilität werden zusätzlich verschiedene browserspezifische Eigenschaften verwendet.

  • Eigentlich sollte es möglich sein, das gleiche Video mehrmals auf einer Seite anzuzeigen. Aber das funktioniert nicht, wenn der Plugin-Aufruf identisch ist, z.B.

    {{{PLUGIN:video('LotR');}}}
{{{PLUGIN:video('LotR');}}}

    Dies wird durch die Skript-Evaluierung des Plugin-Aufrufs verursacht, die identische Aufrufe nur ein einziges mal evaluiert (unter der Annahme, dass die Funktion keine Nebenwirkungen hat). Verwenden Sie in diesem Fall das klassische CMSimple-Skripting, hi_pd_scripting oder überlisten Sie Plugin-Aufruf durch Einfügen eines oder mehrerer Leerzeichen:

    {{{PLUGIN:video('LotR');}}}
{{{PLUGIN:video( 'LotR');}}}

    Dieses Problem ist seit CMSimple_XH 1.6 behoben, so dass Sie bedenkenlos den gleichen Pluginaufruf mehrfach auf einer einzigen Seite notieren können.

  • Die ausgelieferten Skins "tube" und "tube2" sind etwas fehlerhaft.
