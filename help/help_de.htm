Video_XH - ein CMSimple_XH-Plugin

Video_XH ermöglicht die Präsentation von Videos auf einer CMSimple-Website. Es verwendet wo immer möglich das HTML5 Video-Element, und nutzt andernfalls den Flash-Player. Die unterstützten Video-Formate hängen vom Browser bzw. dem Flash-Player ab.

Voraussetzungen

Video_XH ist ein Plugin für CMSimple_XH. Es benötigt eine UTF-8 kodierte Version.

Installation

The Installation erfolgt wie bei vielen anderen CMSimple_XH-Plugins auch. Im CMSimple_XH-Wiki finden Sie weitere Informationen.

  1. Sichern Sie die Daten auf Ihrem Server.
  2. Entpacken Sie das herunter geladene Archiv auf Ihrem Computer.
  3. Laden Sie das komplette Verzeichnis video/ auf Ihren Server in das CMSimple_XH-Pluginverzeichnis hoch.
  4. Machen Sie die Unterverzeichnisse config/, css/ und languages/ beschreibbar.
  5. Gehen Sie zu "Video" im Administrationsbereich, um zu prüfen, ob alle Voraussetzungen erfüllt sind.

Einstellungen

Die Plugin-Konfiguration wird wie bei vielen anderen CMSimple_XH-Plugins auch im Administrationsbereich durchgeführt. Wählen Sie "Plugins" → "Video".

Sie können die Voreinstellungen von Video_XH unter "Konfiguration" ändern. Hinweise zu den Optionen werden angezeigt, wenn Sie die Hilfe-Icons mit der Maus überfahren.

Die Lokalisierung wird unter "Sprache" vorgenommen. Sie können dort die Sprachtexte in Ihre eigene Sprache übersetzen, oder diese gemäß Ihren Wünschen anpassen.

Das Aussehen von Video_XH kann unter "Stylesheet" angepasst werden.

Verwendung

Es ist wichtig, dass der Server die Videos mit dem richtigen Mime-Typ ausliefert. Sollte dies nicht der Fall sein, dann können Sie einen Apache-Server vermutlich mit den folgenden Zeilen in .htaccess entsprechend konfigurieren:

AddType video/webm .webm
AddType video/mp4 .mp4
AddType video/ogg .ogv

Wenn dies nicht möglich ist, bitten Sie Ihren Provider oder Server-Administrator die nötige Konfiguration vorzunehmen.

Da Video_XH keinen Upload anbietet, müssen Sie Ihre Video-Dateien in den eingestellten Order per FTP, Filebrowser oder Uploader_XH hoch laden. Es werden die folgenden Video-Formate unterstützt: MP4 (AVC Baseline@L3.0/AAC), WebM (VP8/Vorbis) und OGG (Theora/Vorbis). Um alle bedeutenden Browser zu unterstützen sollten Sie eine *.mp4 und eine *.webm/*.ogv Version hoch laden. Zusätzlich können Sie ein "Poster" in den gleichen Ordner hoch laden, also ein Bild im JPEG-Format (*.jpg), das angezeigt wird bevor das Video abspielt.

Um ein Video auf einer CMSimple-Seite anzuzeigen, fügen Sie dort den folgenden Plugin-Aufruf ein:

{{{PLUGIN:video(%NAME%, %OPTIONEN%);}}}

Die Bedeutung der Parameter:

%NAME%
Der Name der Video-Datei ohne Datei-Erweiterung. Um genau zu sein, ist dies ein Dateipfad relativ zum eingestellten Video-Ordner.
%OPTIONEN%
Optionen für dieses Video als Zeichenkette im selben Format wie ein Query-String (alles hinter dem Fragezeichen in einem HTTP-GET-Request). Optionen, die nicht angegeben werden, verwenden die entsprechenden Werte aus der Plugin-Konfiguration. Sie können den "Aufruf-Assitenten", der in der Plugin-Administration verfügbar ist, verwenden, um den Aufruf zusammen zu stellen. Nachdem Sie mit der Eingabe der gewünschten Werte fertig sind, kopieren Sie einfach den Inhalt des "Plugin-Aufruf" Eingabefelds und fügen ihn auf einer Seite ein.

Beispiele

{{{PLUGIN:video('LotR');}}}
Dies wird das LotR.webm, LotR.mp4 oder LotR.ogv Video im eingestellten Video-Ordner mit der konfigurieren Breite und Höhe anzeigen. Wenn LotR.jpg existiert, wird es als Poster verwendet.
{{{PLUGIN:video('music/thriller', 'width=320&height=240');}}}
Wird das "thriller" Video aus dem Unterordner music/ mit einer Größe von 320px x 240px anzeigen. Beachten Sie, dass das Skalieren beim Playback nicht unbedingt die beste Option ist.
{{{PLUGIN:video('banner', 'autoplay&loop&controls=0');}}}
Wird automatisch das "banner" Video in einer Schleife abspielen, ohne Steuerelemente anzuzeigen. Das ist besonders nützlich für Banner- oder Hintergrundvideos.

Fehlerbehebung

Falls das Video nicht abgespielt werden kann:

  • Prüfen Sie, ob wenigstens eine passende Video-Datei existiert.
  • Prüfen Sie, ob dieses Video im Browser abgespielt werden kann, wenn Sie es direkt über die URL aufrufen. Das funktioniert allerdings nur, wenn der Browser das Video-Format nativ unterstützt.

Wenn das Video nicht direkt abgespielt werden kann, liegt das vermutlich an einem Kodierungs- oder Multiplex-Fehler. Es scheint, dass viele MP4-Muxer das "moov" Atom ans Ende des Videos hängen, was den Download der gesamten Datei erforderlich macht, bevor die Wiedergabe starten kann. Es ist das Beste zu versuchen, das Video mit Yamb (MP4) oder MKVToolNix (WebM) erneut zu multiplexen.

Einschränkungen

Das HTML5 video Element ist in HTML 4.01 und XHTML 1.0/1.1 ungültig. Falls nötig, sollten Sie also die Doctype-Deklaration in Ihrem Template ändern:

<!DOCTYPE html>

VideoJS' CSS validiert nicht. Grundsätzlich ist es CSS 3, aber für maximale Browser-Kompatibilität werden zusätzlich verschiedene browserspezifische Eigenschaften verwendet.

Eigentlich ist es kein Problem, das gleiche Video mehrmals auf einer Seite anzuzeigen. Aber das funktioniert nicht, wenn der Plugin-Aufruf identisch ist, z.B.

{{{PLUGIN:video('LotR');}}}
{{{PLUGIN:video('LotR');}}}

Die wird durch die Skript-Evaluierung des Plugin-Aufrufs verursacht, die identische Aufrufe nur ein einziges mal evaluiert (unter der Annahme, dass die Funktion keine Nebenwirkungen hat). Verwenden Sie in diesem Fall das klassische CMSimple-Skripting, hi_pd_scripting oder überlisten Sie Plugin-Aufruf durch Einfügen eines oder mehrerer Leerzeichen:

{{{PLUGIN:video('LotR');}}}
{{{PLUGIN:video( 'LotR');}}}
